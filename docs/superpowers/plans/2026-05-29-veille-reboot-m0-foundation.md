# Veille Reboot — M0 Foundation Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Stand up a fresh Veille web app — new git repo, pnpm monorepo, Next.js 15 + better-auth (email magic link) + PostgreSQL/Drizzle — with `@veille/core`, the four adapters, and `@veille/discovery` ported and building, ending at a signed-in empty "your dossiers" home.

**Architecture:** A pnpm workspace (`packages/*` + `apps/web`). The ported packages stay storage-agnostic and keep the universal `extract → Fact[]` contract. `apps/web` is the only thing that knows SQL: Drizzle owns the schema (`dossiers`/`sources`/`facts` + better-auth's tables) against Postgres. Auth is better-auth with the Drizzle adapter and a Resend-backed magic-link flow. No dossier *features* yet — this milestone proves the foundation builds, authenticates, and that the ported pipeline runs in the new repo.

**Tech Stack:** TypeScript 5.6 (strict, ESM/NodeNext), pnpm 10, Next.js 15 (App Router) + React 19 + Tailwind v4, better-auth + `@better-auth/cli`, Drizzle ORM + drizzle-kit + `pg`, Resend, vitest, Node 22 LTS.

**Source repo for ports:** `D:/Projects/CODING/veille` (the current project; read-only reference).
**New repo location:** `D:/Projects/CODING/veille-app` (provisional folder name; product is still "Veille"; rename the folder later if desired).

**Decision (M0 simplification):** Port `@veille/core` *verbatim*, including `subject-store.ts` and `operations.ts`, so the build and tests stay green with zero edits. The spec's "drop the file store" is the *end* state; the persistence swap to Postgres happens in the **M1** plan, where `operations.runRefresh` is re-pointed. Leaving the unused file-store in place for M0 is harmless and avoids breaking the build.

**Docs to check at execution time (APIs move):**
- better-auth: https://www.better-auth.com/docs (Drizzle adapter, magic-link plugin, `@better-auth/cli generate`)
- Drizzle: https://orm.drizzle.team/docs (pg-core schema, drizzle-kit generate/migrate)
- Next 15 App Router + `transpilePackages`: https://nextjs.org/docs

- [ ] **Step 3: Write `apps/web/lib/db/app-schema.ts`**

```ts
import { pgTable, text, timestamp, jsonb, real, uuid } from "drizzle-orm/pg-core";
import { user } from "./auth-schema";

export const dossiers = pgTable("dossiers", {
  id: uuid("id").primaryKey(),
  ownerId: text("owner_id").notNull().references(() => user.id, { onDelete: "cascade" }),
  name: text("name").notNull(),
  intent: text("intent").notNull(),
  language: text("language"),
  template: text("template").notNull().default("feed"), // 'profile' | 'chronology' | 'feed'
  cadence: jsonb("cadence"),                             // recorded by planner; NOT acted on in M1
  status: text("status").notNull().default("active"),
  createdAt: timestamp("created_at", { withTimezone: true }).notNull().defaultNow(),
  refreshedAt: timestamp("refreshed_at", { withTimezone: true }),
});

export const sources = pgTable("sources", {
  id: uuid("id").primaryKey(),
  dossierId: uuid("dossier_id").notNull().references(() => dossiers.id, { onDelete: "cascade" }),
  connector: text("connector").notNull(),  // 'youtube'|'web'|'text'|'pdf'|'tavily'|'rss'|'youtube-channel'
  kind: text("kind").notNull(),            // 'standing' | 'item'
  input: jsonb("input").notNull(),
  label: text("label"),
  lastExtractedAt: timestamp("last_extracted_at", { withTimezone: true }),
  createdAt: timestamp("created_at", { withTimezone: true }).notNull().defaultNow(),
});

export const facts = pgTable("facts", {
  id: uuid("id").primaryKey(),
```
